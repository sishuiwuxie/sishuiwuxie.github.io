JavaScript 闭包理解

闭包是函数和声明该函数的词法环境的组合。
是指拥有多个变量和绑定了这些变量的环境的表达式（通常是一个函数），因而这些变量也是该表达式的一部分。

作用域

JavaScript有两个范围：全局和局部。 在函数定义之外声明的变量是全局变量，它的值可在整个程序中访问和修改。 在函数定义内声明的变量是局部变量。

为了能够完全理解JavaScript的工作原理你需要像引擎一眼思考，从他们角度提出问题，并从他们角度回答这些问题。

编译器会分解成如下词法单元
var 、 a、=、 2、；（空格取决于语法）

执行过程：
b的作用域是全局
foo的作用域是一个函数。
a通过形式参数赋值
console是内置对象
log()引用a的赋值，b的赋值

eval()函数可计算某个字符串，并执行其中的的 JavaScript 代码。

with 将代码的作用域设置到一个特定的作用域中

前边不加 var，默认全局变量（非严格模式）

let是更完美的var

http://www.infoq.com/cn/articles/es6-in-depth-let-and-const

使用var声明的变量，其作用域为该语句所在的函数内，且存在变量提升现象；
使用let声明的变量，其作用域为该语句所在的代码块内，不存在变量提升；
使用const声明的是常量，在后面出现的代码中不能再修改该常量的值。

先有蛋（声明）后有鸡（赋值）

闭包

一个拥有许多变量和绑定了这些变量的环境的表达式（通常是一个函数），因而这些变量也是该表达式的一部分。
闭包无处不在，你需要能够识别并拥抱它。

bar()具有一个涵盖foo()作用域的闭包
bar()被封印在foo()内部

jiuwei()可以访问mingren()内部，然后将jiuwei()作为mingren一个值进行传递，在这个例子中我们将jiuwei()当作返回值。
在mingren()执行后，jiuwei()将值赋给weishouyu();
weishouyu()执行后，mingren()内部应该被销毁，毕竟是九尾（手动滑稽），闭包的会阻止销毁，jiuwei()依然存在
jiuwei()一直在mingren()中存活，是mingren()作用域的闭包，jiuwei()依然持有对该作用域的引用，而这个引用就是闭包。

将 timer（）传递给setTimeout().
timer()是wait作用域的闭包，所以一直存在，所以还保有对变量message的引用。
wait()1000毫秒之后执行，依然存在。

####这就是闭包
