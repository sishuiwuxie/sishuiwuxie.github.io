技术介绍

  • Hexo，基于nodejs静态博客生成系统

  • Github Pages，Github是开源的代码版本控制托管系统，Github Pages是其主页

安装过程

1. Git注册

  • 注册Github,
  • 新建new repository
  • 起名必须和你用户名相同 ### xxxx.github.io
    例：你的用户名为 wangerxiao ，故名为 wangerxiao.github.io
  • 点击 Create repository 就 ok了

    2. Git安装

  • Git官网自行下载默认安装
  • 找到Git Bash 输入一下命令
    git config –global user.name “你的GitHub用户名”
    git config –global user.email “你的GitHub注册邮箱”
    生成密钥
    ssh-keygen -t rsa -C “你的GitHub注册邮箱”
    默认Enter，找到 id_rsa.pub Ctrl+A 全部复制
  • 打开GitHub_Settings_keys 页面，

    新建new SSH Key 随机 title，将key全部粘入，保存Ok

    3. Node安装

  • Nodejs下载 默认安装

    4. Hexo 生成

  • Win+R 输入cmd
  • npm 安装hexo 区别在于是否全局安装
    npm install hexo-cli -g 或 npm install hexo-cli
  • 定位到D盘
    D:
  • 定位到nodejs文件夹，文件夹随意
    cd nodejs
  • 初始化复制文件，并创建 blog文件夹
    hexo init blog
  • 定位到blog文件夹
    cd blog
  • 安装启动服务
    npm install
  • 访问 localhost:4000
    hexo server

    5. Hexo 上传Git

  • 打开站点的配置文件_config.yml,
  • 翻到最后并补充
    deploy:
    type: git
    repo: 这里填入你之前在GitHub上创建仓库的完整路径，记得加上 .git
    branch: master
  • 最后安装Git部署插件，输入命令：
    npm install hexo-deployer-git –save

    6. 绑定域名

  • 阿里云域名注册，实名认证等，此处不再详表
  • 添加三条解析记录，
  1. 151.101.229.147 A
  2. 192.30.252.153 A 是GitHub的地址，你也可以ping你的 http://xxxx.github.io 的ip地址，填入进去。
  3. http://xxxx.github.io CNAME

  • 登录GitHub，进入之前创建的仓库，点击settings，设置Custom domain，输入你的域名
  • 本地博客文件夹 ，进入blog/source目录下，创建一个记事本文件，输入你的域名,最好不要带www.

    6. 重复指令

    hexo clean
    hexo g
    hexo d
    Ok 大功告成
